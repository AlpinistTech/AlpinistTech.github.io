Course Overview

Lab 0

This Lab focuses on setting up a development environment using Ubuntu 22.04 as a Virtual machine. The main focus was to install essential software such as apache web server, Git and necessary tools for the web development. The Lab covers working with Github repositories where we work on cloning, editing, committing and pushing changes. The purpose of this lab is to prepare a solid foundation for web application programming and hacking activities throughout the course.

Lab 1

Overview:
Lab 1 focused on foundational concepts of the web specifically the HTTP protocol and basic web application programming. It was divided into two main parts:

  1. Web and HTTP Protocol
    Hands-on activities involved using Wireshark to capture and analyze HTTP traffic and Telnet to manually send HTTP requests. This section aimed to thoroughly understand how HTTP requests and responses work at the packet level.
  2. Basic web application programming
    Introducing web application development using CGI with C programming and basics of PHP for developing simple web applications that handle user input.

Lab 2

Overview:
In Lab 2, the lecture introduced front-end web development using HTML and JavaScript. It includes concepts, hands-on exercises, and a lab focused on creating a simple dynamic webpage. Key topics covered:

  1. Basic HTML tags to create and deploy simple static HTML pages.
  2. Adding images and forms with GET and POST methods.
  3. Introduction to client-side JavaScript and its role in dynamic web pages, including inline JavaScript.
  4. Hands-on experience with Lab 2.1 and Lab 2.2.

Hackathon 1

In this hackathon, I explored multiple levels of reflected cross-site scripting (XSS) attacks on a simulated vulnerable web application, followed by implementing defense mechanisms through input validation and output encoding. This hands-on experience increased my understanding of both exploiting and mitigating XSS vulnerabilities. Key outcomes included successful script injections displaying my name with alert(), identifying vulnerable backend PHP code, and applying proper data sanitization to secure the application.

This hackathon highlighted the importance of secure coding practices, especially in handling untrusted inputs. By performing real XSS attacks and then fixing the vulnerabilities, I better understood how small oversights can lead to major security breaches. Techniques like htmlentities() in PHP and frontend validation/encoding strengthened the security posture of the app.

Individual Project 1

This individual project involves designing and implementing a secure and functional web application that demonstrates key skills learned throughout the course. It integrates concepts from labs and hackathons, including secure coding, front-end development, and back-end integration. The project showcases my understanding of web technologies, security best practices, and real-world application development.
